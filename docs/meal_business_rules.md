# 餐單業務規則

本文整理程式目前用來決定「一餐是否可見、餐廳午餐、米類備註、小食固定」的規則。主要實作在 `meal_planner/meal_schedule.py`、`meal_planner/schedule_grid.py`，可調設定在 `config.yaml`。

## 1. 更碼命中

- 飯時表 A:E 讀取欄位：`更碼`、`早餐`、`午餐`、`小食`、`晚餐`。
- 更碼規則由上而下第一個命中：
  - `ABC` 只命中完全相同的更碼。
  - `ABC*` 命中以前綴 `ABC` 開頭的更碼。
  - `其他` 不參與第一輪命中，只作兜底。
- 行位表更碼較嚴格：
  - 完全相同會命中。
  - `PenC(…)` 或 `PenC（…）` 會命中 `PenC`。
  - `PenC頂位` 不會當成 `PenC`。

### 行位表版本生效日期

行位表可加 optional 欄位 `生效日期`（亦接受 `生效` / `Effective From`）。

- 同一更碼可以有多套行位表 rows。
- 產生某日餐單時，只會使用「生效日期小於或等於該日」之中最新一套。
- 如果該日未有任何已生效版本，會使用沒有 `生效日期` 的舊版 rows。
- 例：`EleC1` 新 rows 填 `2026-06-01`，則 2026-06-01 起用新版；2026-05-31 或之前仍用未填生效日期的舊版。

## 2. 一餐是否可見

後端產生餐單時，會先用飯時命中行決定該日理論上有哪幾餐，再只為可見餐取 Pattern。

有日期時：

- 優先用行位表、加班表和飯時規則解析 `meal_times_resolved`。
- 解析後某餐有非空時間字串，該餐可見。
- `跟行位表` 只支援：
  - 午餐：取行位表第一個內容包含 `飯` 的時間。
  - 小食：取行位表第一個內容包含 `小食` 的時間。
- `開工前 N 小時` 用報開工時間倒推；沒有 N 時預設 2 小時。
- `收工後 N 小時` 用報收工時間順推；沒有 N 時預設 1.5 小時。
- 加班表同日 `開工` / `收工` 會覆蓋行位表報開工 / 報收工。

沒有日期或無法解析時間時：

- 回退用飯時 A:E 命中行本身：某餐欄位非空即視為可見。

前端顯示時：

- 若 `meal_times_resolved` 有非空值就顯示該餐。
- 否則只接受飯時原文是 `HH:MM` 的固定時間。
- 因此 `跟行位表` 若後端未解析到實際時間，前端不會顯示該餐。

## 3. Pattern 來源

- Pattern 來自飯時表 G:H 的獨立表，欄位為 `餐名` / `Pattern`。
- A:E 只決定時間與可見餐，不再按同一行綁定 Pattern。
- Pattern 分隔：
  - `+` 分隔食物項目。
  - `/` 分隔同一項目的候選替代。
  - 兩者可在 `config.yaml` 的 `pattern` 設定。

## 4. 餐廳午餐

- 餐廳選擇表以 `更碼關鍵字` 命中更碼，命中規則與飯時更碼相同。
- 預設只在返工日套用，設定為：

```yaml
meal_business_rules:
  restaurant_lunch_workday_only: true
```

- 命中餐廳午餐後：
  - 午餐內容顯示餐廳選擇。
  - 午餐營養值使用餐廳選擇表該行的固定營養值。
  - 午餐不再交由 solver 配食材。
  - 午餐 `meal_items` 清空，所以米類備註不會把餐廳午餐當成米飯熟重來源。
- 餐廳午餐營養會作為固定已食用營養，其他餐再遷就它。

## 5. 小食固定

- 預設小食是固定餐，不交由 solver 調整：

```yaml
meal_business_rules:
  fixed_meals: ["小食"]
```

- 固定餐仍要符合「可見」且有 Pattern 才生效。
- 固定餐按候選輪替揀食材與克數，營養值扣入全日總數。
- solver 只負責其餘可調餐次。

## 6. 米類備註

- 同日米類只鎖定一款：配餐時第一個米類候選會被全日重用。
- solver / 輪替判斷米類 Pattern 項目時，用 `rice.rice_category_exact`，預設 `"米"`。
- 米類備註計算已選食材熟重時，用食材名稱包含字串判斷：

```yaml
rice:
  note_name_contains: ["米"]
  brown_name_contains: "糙米"
```

- 食材名稱包含 `brown_name_contains` 時用糙米熟重轉生重比例，否則用其他米比例。
- 備註公式：
  - 生重 = 熟重總和 / 對應熟重轉生重比例
  - 水 = 生重 * `water_multiplier`
- 沒有米類食材時，顯示 `（米類備註：配餐後填）`。

## 7. 目前仍屬程式規則

以下規則已文件化，但暫未抽到設定，因為它們與工作簿欄位語義直接綁定：

- `跟行位表` 午餐找 `飯`，小食找 `小食`。
- 相對時間文字只解析 `開工前 N 小時`、`收工後 N 小時`。
- 前端只把固定 `HH:MM` 原文作為可顯示時間 fallback。
